# 統合開発コマンド

指定されたissueに対して、ブランチ作成から実装、PR作成まで一連の開発フローを実行します。

## 使用方法
```
/develop {issue番号}
```

## 動作内容
1. **issue読み取り**
   - GitHub issueの内容を取得
   - 開発タスクを理解

2. **ブランチ作成**
   - mainから最新を取得
   - feat/issue-{番号}-{説明}形式でブランチ作成

3. **実装**
   - issueの要件に基づいて実装
   - 適切な粒度でコミット
   - lint・formatチェックを都度実行

4. **PR作成**
   - 全ての変更をpush
   - PRを作成してURLを返す

## 例
```
/develop 5
```
→ issue #5を自動で実装してPRまで作成

## 注意事項
- 実装中に判断に迷う場合は都度確認
- セキュリティに関わる変更は必ず確認
- 大規模な変更の場合は段階的に確認
